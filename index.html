Making Android App in Windows Shell

Setup

  • Download JDK1.6 means Java Development Kit Version 1.6.Install it.
  • Download JRE1.6 means Java Runtime Environment. Install it.
  • Next, select any directory where you will make your project.
  • In that directory, make 5 sub-directories:- "java","class","plat","dex","res".
  • Concept is that you will make java programs as source code and keep in 'java' directory.
  • Use 'aapt' command to read 'res' directory and generate corresponding java program 'R.java' again in same 'java' directory.
  • Using 'javac' command you will compile all java programs into ".class" files and keep in "class" directory.
  • While compiling you may need external libraries in '.jar' files which are also compressed '.class' files. So keep them also in 'class' directory.
  • Compulsory external library .jar file is android.jar of respective android version you target. We keep it in 'plat' directory separately for 'aapt' command
  • Then you will use 'dx' command to combine all class files into .dex file and keep in 'dex' directory.
  • Lastly you will make an '.apk' file fullfrom : application package, by packing '.dex','.class' & files in 'res' directory together, using 'aapt' again.
  • Point is that 'dex' file is dalvic executable which will run on android operating system in mobile.
  • And that program will use images, sounds and other resource files from 'res' directory.
  • Using 'adb' command, you transfer the apk file app to ur mobile using usb data cable

Commands

  • aapt p -f -m -M AndroidManifest.xml -I plat/android.jar -S res -J java
  • Above translates to:- Android App Package Tool(aapt) pack(p) , make directory if not exist (-m),force file overwrite (-f), (-M) manifest file is: AndroidManifest.xml , (-I) include the files from platfom/android.jar , (-S) reSource files in 'res' directory , (-J) generate Java file in package's directory com.pkg as given in AndroidManifest.xml file.
  • javac java/com/pkg/*.java -classpath class;pkg/android.jar;class/file1.jar;class/file2.jar; -d class
  • Above translates to:- java-compile (javac) the .java files in package-directory and when you encounter new class variable , then find them in class-paths i.e. in directory 'class'; or 'within zipped jar files of android.jar & external library file1.jar file1.jar etc. and finally put the compiled classes again in 'class' directory itself.
  • dx --dex --output=dex/classes.dex class class/file1.jar class/file2.jar
  • Above translates to:- dalvic executable command(dx) --dex means make a dex file output into directory 'dex' onto file 'classes.dex' by combining all classes
  • aapt p -v -f -m -M AndroidManifest.xml -I plat/android.jar -S res -F app.apk dex
  • Above translates to once again use aapt but this time to create .apk file by packaging .dex,android.jar respective platform or version classes, 'res' directory and gather needed package directory location, permissions,hardware-features, activity, service etc. classes' names & intents & entry points from manifest file (-F) means filename
  • adb install app.apk
  • For above to work adb driver for your system, has to be installed.
