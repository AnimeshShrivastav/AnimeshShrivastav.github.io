Defining Java Class Files to Run Upon Launching An App by User on mobile, inside "AndroidManifest.xml" file

Earlier we saw sample java program which is shaped like an activity-class.
A Sample "AndroidManifest.xml" File

<?xml version="1.0" encoding="utf-8"?> 

<manifest xmlns:android="http://schemas.android.com/apk/res/android" 
package="com.pkg" android:versionCode="1"  android:versionName="1.0">

<uses-permission android:name="android.permission.INTERNET" />

<application android:label="Chhotahathee" android:icon="@drawable/icon"> 
        <activity  android:name="MainActivity">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
         </intent-filter>
        </activity>
    </application>
</manifest>
Package attribute above signifies the sub directory where actual java source code resides. And also where 'javac' compiler command will store the generated compiled 'class' files within the (-d)destination directory. Next, we see inclusion of resources.
