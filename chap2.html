Defining Java Class Files to Run Upon Launching An App by User on mobile, inside "AndroidManifest.xml" file

Earlier we saw sample java program which is shaped like an activity-class.
A Sample "AndroidManifest.xml" File

<?xml version="1.0" encoding="utf-8"?> 

<manifest xmlns:android="http://schemas.android.com/apk/res/android" 
package="com.pkg" android:versionCode="1"  android:versionName="1.0">

<uses-permission android:name="android.permission.INTERNET" />

<application android:label="MyApp" android:icon="@drawable/icon"> 
        <activity  android:name="MainActivity">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
         </intent-filter>
        </activity>
    </application>
</manifest>
Line By Line Translation of above file:-
  • Use XML Language version 1.0 to interpret this xml file. Also interpret characters as Universal Text Code-8 bit chunks ASCII etc
  • xml name-space or xmlns is the prefix to add to all variables encountered here in this xml file, just to make them uniquely understood by android.jar. Also now onwards android can be used as a short form rather than such big url type prefix.
  • Package attribute above signifies the sub directory where actual java source code resides. And also where 'javac' compiler command will store the generated compiled 'class' files within the (-d)destination directory.
  • Version Code and Name together constitute latest update of your app like version 1.0 earliest, then 1.1 next update etc.
  • Permission is to be prompted to user during installation, say from google play store.
  • android:name & android:label etc. will define variables name , label,etc. for use in each tagged line of xml statement
  • label & icon variables are defined for application tag of xml
  • Similarly each activity's name is defined under variable android:name under activity tag statement of xml
  • Then name & category variables are defined for intent statement of xml.
  • Intent of MAIN action will define the main entry point i.e. associated activity will be created first by operating system of mobile
  • Category name LAUNCHER will mean that when mobile's inbuilt launcher is used to launch or start the app.
